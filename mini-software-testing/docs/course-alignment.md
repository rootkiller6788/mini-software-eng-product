## Industry References

- **xUnit Patterns** (Meszaros) — Test double patterns (Stub, Mock, Fake, Spy)
- **Working Effectively with Unit Tests** (Jay Fields) — Test design principles
- **Mutation Testing** (Jia & Harman 2011) — Survey of mutation analysis
- **Performance Testing Guidance** (Microsoft Patterns & Practices)
- **JUnit XML Format** — De facto standard for test result interchange

## Module Coverage

1. **Test Framework** — Test runner with suites/cases, assertion recording, pass/fail/error/skip tracking
2. **Test Doubles** — Stubs (fixed return), Mocks (expected calls), Fakes (lightweight real), Spies (call recording)
3. **Code Coverage** — Statement, branch, and MC/DC coverage tracking per file
4. **Performance** — Benchmark recording (min/max/avg/p50/p95/p99), regression detection, load profiles
5. **Test Reporting** — Run history, pass rate trends, flaky test detection, JUnit XML export, mutation testing
